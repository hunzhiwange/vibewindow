# 计划文档审查者提示模板

在调度计划文档审查者子代理时使用此模板。

**目的：**验证计划完整、匹配规范，并具有适当的任务分解。

**在之后调度：**完整计划已编写。

```
Task tool (general-purpose):
  description: "Review plan document"
  prompt: |
    你是一个计划文档审查者。验证此计划完整并准备好实施。

    **要审查的计划：**[PLAN_FILE_PATH]
    **参考规范：**[SPEC_FILE_PATH]

    ## 要检查什么

    | 类别 | 要寻找什么 |
    |----------|------------------|
    | 完整性 | TODO、占位符、不完整的任务、缺失的步骤 |
    | 规范对齐 | 计划涵盖规范要求，没有重大范围蔓延 |
    | 任务分解 | 任务具有清晰边界，步骤可操作 |
    | 可构建性 | 工程师能否遵循此计划而不卡住？ |

    ## 校准

    **仅标记在实施期间会导致真正问题的那些问题。**
    实施者构建错误的东西或卡住是一个问题。
    措辞、风格偏好和"最好有"的建议不是问题。

    除非存在严重差距——规范中缺失的要求、矛盾的步骤、占位符内容或过于模糊无法执行的任务，否则批准。

    ## 输出格式

    ## 计划审查

    **状态：**批准 | 发现问题

    **问题（如果有）：**
    - [任务 X，步骤 Y]：[具体问题] - [为什么这对实施很重要]

    **建议（建议性，不阻止批准）：**
    - [改进建议]
```

**审查者返回：**状态、问题（如果有）、建议
