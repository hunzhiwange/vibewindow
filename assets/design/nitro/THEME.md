# Nitro 风格提示

## 一句话定义

Nitro = 专业、秩序、理性、克制，适合企业官网、B 端产品页与服务介绍页的稳定型 Light 设计系统。

## 视觉气质

- 背景干净，结构稳定，重点靠网格、描边、层级和留白建立可信感。
- 主色偏业务蓝，强调“可靠、清楚、可执行”，不是消费级情绪化转化风。
- 卡片、表单、表格、侧边栏要规整克制，不做 Halo 式圆润柔软感。
- 允许少量状态色，但状态色只服务反馈，不主导整页视觉。

## 版式规则

- 页面优先单栏垂直流，模块之间保持明确分段。
- 首屏优先“标题 + 说明 + 主次 CTA + 证据块 / 产品面板”。
- 中段适合“能力矩阵 / 方案分组 / 指标卡 / 流程说明 / 表格对比”。
- 收口适合“FAQ / 联系表单 / 服务咨询 / 下一步动作”。
- 模块内部优先“标题区 + 内容区 + 动作区”，不要用大量漂浮装饰打散信息。

## 组件语义

- Button：直角、清晰、分层明确，主按钮强调业务动作，Outline 用于低强调动作。
- Label：用 `Label/*` 做状态标签或分类标签，不做 Halo 式软胶囊 badge。
- Input / Select / Textarea：边界清楚，适合收资、筛选、后台输入。
- Card：承载指标、方案、案例、摘要，优先直角卡片与细描边。
- Table / Data Table：用于结构化数据、方案对比、后台列表。
- Sidebar：用于工作台、后台、管理视图，不默认带入官网首页。
- Alert / Tooltip：分别承担状态反馈与短提示，不替代正文内容容器。

## 排版规则

- 主字体使用 `$--font-primary`，默认是更中性的业务字重与字面。
- 数据、编号、筛选值、技术字段可用 `$--font-secondary` 强化秩序感。
- 标题直接、明确、少修辞，避免情绪化营销措辞。
- 正文、说明、注释优先使用 `$--muted-foreground` 控制层级。
- 表格、表单、列表默认左对齐，数字与指标模块允许更规整的列式排布。

## 常用 token

- 背景：`$--background`
- 主文字：`$--foreground`
- 卡片：`$--card`
- 卡片文字：`$--card-foreground`
- 主操作：`$--primary`
- 主操作文字：`$--primary-foreground`
- 次操作块：`$--secondary`
- 弱化说明：`$--muted-foreground`
- 描边：`$--border`
- 输入描边：`$--input`
- 侧边栏：`$--sidebar`
- 侧边栏激活：`$--sidebar-primary`
- 信息状态：`$--color-info`
- 成功状态：`$--color-success`
- 警告状态：`$--color-warning`
- 错误状态：`$--color-error`
- 默认圆角：`$--radius-none`
- 悬浮提示圆角：`$--radius-xs`

## 风格继承 JSON 说明

- `nitro/theme` 只保留 Nitro 风格继承真正需要的核心变量。
- 变量覆盖背景、前景、卡片、按钮、表单、侧边栏、状态色、字体、圆角等关键语义。
- `nitro/components` 只保留 Nitro 最关键的基础组件骨架、状态变体与继承关系。
- `nitro/DESIGN.md` 提供生成白名单、最小结构与复用规则。
- 如果希望 AI 不注入完整 `nitro.json` 也能稳定继承 Nitro，应优先注入 `nitro/THEME.md`、`nitro/DESIGN.md`、`nitro/theme.json`、`nitro/components.json`。

## 适合的模块

- 企业首屏
- 解决方案分组
- 功能与能力矩阵
- 服务流程说明
- 套餐对比 / 方案表格
- 客户案例 / 行业场景
- FAQ
- 联系表单 / 预约咨询
- 产品后台 / 控制台 / 数据概览

## 不适合的模块

- 高情绪化消费级活动页
- 软萌圆润的轻转化落地页
- 强未来感暗黑科幻首页
- 依赖大面积装饰图形的品牌展示页
- 大量漂浮信息块和无网格的实验性排版
