# Shadcn 风格提示

## 一句话定义

Shadcn = 中性、极简、克制、现代通用产品站。

## 视觉气质

- 默认偏 Light，中性背景、轻描边、清晰正文对比。
- 重点靠布局、排版和容器秩序建立品质感，而不是靠装饰情绪。
- 整体要干净、稳、可组合，适合作为大多数 Web 产品的默认审美基线。
- 不做 Halo 的转化氛围，不做 Nitro 的企业秩序感，不做 Lunaris 的未来暗色科技感。

## 版式规则

- 页面通常为单栏垂直流，模块分段清楚。
- 首屏优先“标题 + 描述 + 主次 CTA + 轻视觉辅助”。
- 中段适合功能卡片、能力列表、内容摘要、FAQ、表单收口。
- 栅格可以用 2 列或 3 列，但每列内容都要保持呼吸感。
- 模块之间靠留白与轻边框拉层级，不靠高饱和背景切块。

## 组件语义

- Button：中等圆角，主按钮深色或高对比色，次按钮保持中性。
- Input / Textarea / Select：统一边框、统一高度节奏，强调可读与可用。
- Card：简洁承载容器，适合功能说明、摘要、配置块。
- Badge：轻标签、状态标签，不承载动作。
- Tabs：模块内切换，不做整站导航。
- Alert：状态提示与反馈，不代替正文内容区。
- Sidebar：更适合工作台或后台文档场景，官网默认少用。

## 排版规则

- 标题直接、明确、少修辞。
- 正文用短句和可扫读结构，避免过度口号化。
- 次级说明优先使用 `$--muted-foreground`。
- 字体统一走 `$--font-primary` / `$--font-secondary`。
- 大标题可以有对比，但正文与控件文字要保持克制稳定。

## 常用 token

- 背景：`$--background`
- 主文字：`$--foreground`
- 卡片：`$--card`
- 卡片文字：`$--card-foreground`
- 主操作：`$--primary`
- 主操作文字：`$--primary-foreground`
- 次级块：`$--secondary`
- 弱化块：`$--muted`
- 弱文字：`$--muted-foreground`
- 描边 / 输入：`$--border` / `$--input`
- 侧栏：`$--sidebar`
- 圆角：`$--radius-s` / `$--radius-m` / `$--radius-l`
- 字体：`$--font-primary` / `$--font-secondary`

## 风格继承 JSON 说明

- `shadcn/theme.json` 只保留继承 Shadcn 风格真正需要的核心 token。
- 该 JSON 保留默认 `Light` 模式，并附带必要的 `Dark` 对照值，方便 AI 保持同一套语义层级。
- `shadcn/components.json` 只保留最常用的基础组件骨架、状态变体、继承关系和覆盖规则。
- 生成时优先注入 `THEME.md + DESIGN.md + theme.json + components.json`，不直接依赖完整 `shadcn.json`。
- 如果需要复用结构，看 `components.json`；如果需要控制合法输出和复用方式，看 `DESIGN.md`；如果需要确定风格语气与 token 倾向，看当前文件与 `theme.json`。

## 适合的模块

- 现代产品官网首屏
- 功能 / 能力卡片区
- 产品特性对比区
- 内容摘要区
- FAQ / Accordion
- 文档入口与帮助中心摘要
- 表单收口、联系区、订阅区

## 不适合的模块

- 强营销转化页、强情绪活动页
- 重装饰品牌故事页
- 未来科技暗色炫光页面
- 高密度监控大屏
- 首页即大型表格和复杂数据报表

## 快速生成口诀

如果用户没有给出更强风格指令，默认按以下组合生成：

- Light 模式
- 中性浅背景
- 深色主按钮
- 轻边框卡片
- 中等圆角
- 清晰标题层级
- 平衡信息密度
- 可组合的现代产品站语气
