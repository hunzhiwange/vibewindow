# Halo 风格提示

## 一句话定义

Halo = 明亮、友好、圆润、轻转化、留白宽松。

## 视觉气质

- 整体偏 Light，背景干净，信息层级轻盈。
- 主视觉不冷硬，允许轻情绪感与轻渐变，但不要花哨。
- 强调“易读 + 易点 + 易转化”，不是“企业后台”或“重数据面板”。

## 版式规则

- 页面通常为单栏垂直流，模块分块明显。
- 首屏优先“标题 + 描述 + CTA + 视觉焦点”。
- 中段适合“3 列或 2 列卡片卖点”。
- 结尾适合“评价 / FAQ / 联系 CTA / 订阅收口”。
- 留白要比普通企业站更宽松，模块之间保持清晰呼吸感。

## 组件语义

- Button：优先胶囊形，主按钮用 `$--primary`。
- Badge：小尺寸、全圆角、轻提示。
- Input / Textarea：圆角更大，边框柔和。
- Card：白底、轻描边、轻阴影感，但不要厚重。
- Alert / Tooltip：信息提示友好，不做强压迫感。
- Tabs / Switch / Radio：形态偏圆润，状态反馈清楚。

## 排版规则

- 标题清晰直接，避免过于技术化或冷硬措辞。
- 大标题可稍有情绪感，但正文必须克制、易扫读。
- 正文和说明优先使用 `$--muted-foreground`。
- 字体使用 `$--font-primary` / `$--font-secondary`。

## 常用 token

- 背景：`$--background`
- 主文字：`$--foreground`
- 卡片：`$--card`
- 主按钮：`$--primary`
- 次级块：`$--secondary`
- 弱文字：`$--muted-foreground`
- 描边：`$--border`
- 中圆角：`$--radius-m`
- 胶囊圆角：`$--radius-pill`

## 风格继承 JSON

- `halo/theme` 现在不是只放最小示例，而是放 Halo 风格继承需要的核心变量集合。
- 其中包含背景、文字、卡片、按钮、弱化文本、描边、侧边栏、状态色、字体、圆角等 token。
- 如果希望 AI 在不注入完整 `halo` 的情况下仍继承 Halo 风格，应优先注入这个 `theme`。
- `halo/components` 提供基础组件骨架、状态变体与继承关系。
- 组件骨架看 `halo/components`，复用规则看 `halo/DESIGN.md` 与 `theme-halo-index.md`，颜色与风格继承看 `halo/theme`。

## 适合的模块

- 品牌首屏
- 功能卖点卡片
- 用户评价 / 案例背书
- FAQ
- 联系或订阅收口

## 不要生成成这样

- 不要做冷硬 B 端控制台风。
- 不要做密集表格、重数据图表首页。
- 不要使用大量尖锐直角和高压对比。
- 不要堆满装饰图形导致内容不易读。

## 快速生成口诀

如果用户没有明确指定细节，默认按以下组合生成：

- Light 模式
- 大面积浅背景
- 白色卡片
- 紫色主操作
- 圆角按钮与输入框
- 宽松留白
- 轻转化官网语气
