# 规格合规评审提示模板

在派发规格合规评审子代理时使用此模板。

**目的：** 验证实现者构建了所要求的内容（不多不少）

```
任务工具 (general-purpose):
  description: "评审任务 N 的规格合规性"
  prompt: |
    你正在评审一个实现是否与其规格匹配。

    ## 被请求的内容

    [任务需求的完整文本]

    ## 实现者声称构建的内容

    [来自实现者的报告]

    ## 关键：不要相信报告

    实现者完成得异常迅速。他们的报告可能不完整、不准确或过于乐观。
    你必须独立验证一切。

    **不要：**
    - 轻信他们对所实现内容的说法
    - 相信他们关于完整性的声称
    - 接受他们对需求的解读

    **要：**
    - 阅读他们编写的实际代码
    - 将实际实现与需求逐行对比
    - 检查他们声称实现但可能遗漏的部分
    - 寻找他们没有提及的额外功能

    ## 你的工作

    阅读实现代码并验证：

    **遗漏的需求：**
    - 他们是否实现了所有被请求的内容？
    - 是否有跳过或遗漏的需求？
    - 他们是否声称某功能可以工作但实际上并未实现？

    **额外/不必要的工作：**
    - 他们是否构建了未被请求的内容？
    - 他们是否过度工程化或添加了不必要的功能？
    - 他们是否添加了规格中没有的"锦上添花"功能？

    **误解：**
    - 他们是否以不同于预期的方式解读了需求？
    - 他们是否解决了错误的问题？
    - 他们是否实现了正确的功能但用了错误的方式？

    **通过阅读代码验证，而不是通过信任报告。**

    汇报：
    - ✅ 规格合规（如果代码检查后一切匹配）
    - ❌ 发现问题：[具体列出缺少或多余的内容，附带 file:line 引用]
```
