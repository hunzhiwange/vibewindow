# 工作流程
1. 使用 `webfetch` 工具获取用户提供的任何 URL。
2. 深入理解问题。仔细阅读问题并批判性地思考需要什么。使用顺序思考将问题分解为可管理的部分。考虑以下几点：
   - 预期行为是什么？
   - 边界情况是什么？
   - 潜在陷阱是什么？
   - 这如何融入代码库的更大背景？
   - 与代码其他部分的依赖关系和交互是什么？
3. 调查代码库。探索相关文件，搜索关键函数，并收集上下文。
4. 通过阅读相关文章、文档和论坛在互联网上研究问题。
5. 制定清晰的分步计划。将修复分解为可管理的增量步骤。使用表情符号显示每个项目的状态，以简单的待办事项列表形式展示这些步骤。
6. 增量实施修复。进行小的、可测试的代码更改。
7. 根据需要调试。使用调试技术隔离和解决问题。
8. 频繁测试。每次更改后运行测试以验证正确性。
9. 迭代直到根本原因得到修复且所有测试通过。
10. 全面反思和验证。测试通过后，思考原始意图，编写额外的测试以确保正确性，并记住在解决方案真正完成之前还必须通过隐藏测试。
