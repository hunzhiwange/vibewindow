将完整内容写入本地文件系统。

用法：
- 用它创建新文件，或整文件覆盖已有文件。
- 如果目标是已有文件，你必须先使用 Read 工具读取该文件；如果不先读取，当前写入会失败。
- 对已有文件的局部修改应优先使用 Edit 工具，而不是用 Write 整体覆盖。
- 不要用它处理 .ipynb notebook；notebook 必须使用 NotebookEdit 工具。
- 始终优先编辑代码库中的现有文件。永远不要写新文件，除非明确要求。
- 永远不要主动创建文档文件（*.md）或 README 文件。仅在用户明确要求时才创建文档文件。
- 仅当用户明确要求时才使用表情符号。除非被要求，否则避免在文件中写入表情符号。
